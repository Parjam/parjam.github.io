Ruby Enumerables Cheat Sheet

  • all? [{ |obj| block }] Passes each element of the collection to the given block. The method returns true if the block never returns false or nil.
  • any? [{ |obj| block }] Passes each element of the collection to the given block. The method returns true if the block ever returns a value other than false or nil.
  • chunk { |elt| ... } Enumerates over the items, chunking them together based on the return value of the block. Consecutive elements which return the same block value are chunked together.
  • count, count(item), count { |obj| block } Returns the number of items in enum through enumeration. If an argument is given, the number of items in enum that are equal to item are counted. If a block is given, it counts the number of elements yielding a true value.
  • drop(n), drop_while { |arr| block } Drop(n) drops first n elements from enum, and returns rest elements in an array. Drop_while drops elements up to, but not including, the first element for which the block returns nil or false and returns an array containing the remaining elements.
  • each_cons(n) { ... } Iterates the given block for each array of consecutive elements. If no block is given, returns an enumerator.
  • each_slice(n) { ... } Iterates the given block for each slice of elements. If no block is given, returns an enumerator.
  • each_with_index(*args) { |obj, i| block } Calls block with two arguments, the item and its index, for each item in enum. Given arguments are passed through to each().
  • each_with_object(obj) { |(*args), memo_obj| ... } Iterates the given block for each element with an arbitrary object given, and returns the initially given object.
  • find(ifnone = nil) { |obj| block } Passes each entry in enum to block. Returns the first for which block is not false. If no object matches, calls ifnone and returns its result when it is specified, or returns nil otherwise.
  • find_all { |obj| block } Returns an array containing all elements of enum for which the given block returns a true value.
  • first(n) Returns the first element, or the first n elements, of the enumerable. If the enumerable is empty, the first form returns nil, and the second form returns an empty array.
  • group_by { |obj| block } Groups the collection by result of the block. Returns a hash where the keys are the evaluated result from the block and the values are arrays of elements in the collection that correspond to the key.
  • include?(obj) Returns true if any member of enum equals obj. Equality is tested using ==.
  • inject(initial) { |memo, obj| block } Combines all elements of enum by applying a binary operation, specified by a block or a symbol that names a method or operator.
  • map { |obj| block } Returns a new array with the results of running block once for every element in enum.
  • max, max { |a, b| block } Returns the object in enum with the maximum value. The first form assumes all objects implement Comparable; the second uses the block to return a <=> b.
  • max_by { |obj| block } Returns the object in enum that gives the maximum value from the given block.
  • min { |a, b| block } Returns the object in enum with the minimum value. The first form assumes all objects implement Comparable; the second uses the block to return a <=> b.
  • min_by { |obj| block } Returns the object in enum that gives the minimum value from the given block.
  • minmax, minmax { |a, b| block } Returns two elements array which contains the minimum and the maximum value in the enumerable. The first form assumes all objects implement Comparable; the second uses the block to return a <=> b.
  • minmax_by { |obj| block } Returns a two element array containing the objects in enum that correspond to the minimum and maximum values respectively from the given block.
  • none? [{ |obj| block }] Passes each element of the collection to the given block. The method returns true if the block never returns true for all elements. If the block is not given, none? will return true only if none of the collection members is true.
  • one? [{ |obj| block }] Passes each element of the collection to the given block. The method returns true if the block returns true exactly once. If the block is not given, one? will return true only if exactly one of the collection members is true.
  • partition { |obj| block } Returns two arrays, the first containing the elements of enum for which the block evaluates to true, the second containing the rest.
  • reject { |obj| block } Returns an array for all elements of enum for which the given block returns false.
  • select { |obj| block } Returns an array containing all elements of enum for which the given block returns a true value.
  • sort { |a, b| block } Returns an array containing the items in enum sorted, either according to their own <=> method, or by using the results of the supplied block.
  • sort_by { |obj| block } Sorts enum using a set of keys generated by mapping the values in enum through the given block.
  • take_while { |arr| block } Passes elements to the block until the block returns nil or false, then stops iterating and returns an array of all prior elements.
